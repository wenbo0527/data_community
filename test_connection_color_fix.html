🔧 连接线颜色修复验证报告

🚨 问题描述

现象：保存任务流程后重新加载时，连接线颜色从蓝色变为黑色，与新建连接线的颜色不一致。

颜色对比

状态 颜色 颜色值 说明
保存前（新建连接） 蓝色 #5F95FF 通过 createEdge 方法创建，包含完整样式配置
保存后（恢复连接） 黑色 #000000 通过 addConnectionToGraph 方法创建，缺少样式配置

🔧 根因分析

问题出现在 TaskFlowCanvas.vue 的 addConnectionToGraph 方法中：

1. createEdge 方法（正确的样式配置）

createEdge() { return new Shape.Edge({ attrs: { line: { stroke: '#5F95FF', // ✅ 蓝色配置 strokeWidth: 2, targetMarker: { name: 'block', width: 12, height: 8, }, }, }, zIndex: 0, }) }

2. addConnectionToGraph 方法（缺少样式配置）

const edgeConfig = { id: connectionData.id, source: { cell: connectionData.source, port: connectionData.sourcePort }, target: { cell: connectionData.target, port: connectionData.targetPort }, router: { name: 'orth', args: { ... } }, connector: { name: 'rounded', args: { ... } }, // ❌ 缺少 attrs 样式配置 data: { ... } }

✅ 修复方案

在 addConnectionToGraph 方法的 edgeConfig 中添加与 createEdge 相同的样式配置：

const edgeConfig = { id: connectionData.id, source: { cell: connectionData.source, port: connectionData.sourcePort }, target: { cell: connectionData.target, port: connectionData.targetPort }, router: { name: 'orth', args: { ... } }, connector: { name: 'rounded', args: { ... } }, connectionPoint: { name: 'boundary', args: { anchor: 'center' } }, // 🔧 修复：添加连接线样式配置，确保颜色一致性 attrs: { line: { stroke: '#5F95FF', strokeWidth: 2, targetMarker: { name: 'block', width: 12, height: 8, }, }, }, zIndex: 0, data: { ... } }

🧪 修复验证

修复后效果对比

状态 颜色 颜色值 修复状态
保存前（新建连接） 蓝色 #5F95FF 保持一致

验证步骤

  1. 创建连接：在任务流程中创建新的连接线，观察颜色为蓝色
  2. 保存流程：保存当前任务流程配置
  3. 重新加载：刷新页面或重新打开任务流程
  4. 验证颜色：确认恢复的连接线颜色仍为蓝色
  5. 一致性检查：新建连接和恢复连接的颜色完全一致
