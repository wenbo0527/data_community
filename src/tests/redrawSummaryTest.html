📋 重绘总结日志功能测试

布局方向切换时的详细位置信息统计

🎯 功能概述

在布局方向切换的重绘机制完成时，系统会自动生成一个详细的总结日志，包含：

  • 节点统计：普通节点和拖拽点的数量及位置
  • 端口统计：输入端口和输出端口的位置信息
  • 连接线统计：普通连接和预览线的路由信息
  • 布局质量分析：自动评估对齐质量和间距分布

🔧 实现位置

📁 文件位置

src/composables/useStructuredLayout.js

⚙️ 核心函数

logRedrawSummary(graph, direction)

🔄 调用时机

在 switchLayoutDirection 函数的重绘完成后自动调用

📏 质量评估标准

对齐质量

  • 优秀：节点在主轴方向上对齐良好（偏差 ≤ 50px）
  • 良好：节点对齐基本合理（偏差 ≤ 100px）
  • 需要改进：节点对齐存在明显问题（偏差 > 100px）

间距分布

  • 合理：节点间距适中（最小间距 ≥ 50px，平均间距 ≤ 300px）
  • 可接受：节点间距基本合理
  • 需要调整：节点过于密集或过于分散

整体评分

  • 优秀：综合评分 ≥ 80分
  • 良好：综合评分 ≥ 60分
  • 需要优化：综合评分 < 60分

🚀 使用方法

自动触发

当调用布局方向切换函数时，重绘总结日志会自动生成：

// 切换到从左到右布局 await switchLayoutDirection('LR') // 切换到从上到下布局 await switchLayoutDirection('TB')

手动调用

import { useStructuredLayout } from '@/composables/useStructuredLayout' const { logRedrawSummary } = useStructuredLayout() // 手动生成总结日志 await logRedrawSummary(graph, 'LR')

💡 应用场景

  • 开发调试：帮助开发者了解布局切换后的实际效果
  • 性能监控：监控重绘操作的完成情况
  • 质量评估：自动评估布局质量，提供优化建议
  • 问题排查：当布局出现问题时，提供详细的位置信息用于排查

⚠️ 注意事项

  • 总结日志会在控制台输出，建议在开发环境使用
  • 对于大型画布（节点数量 > 50），日志可能较长
  • 坐标信息已四舍五入到整数，便于阅读
  • 质量分析基于经验值，可根据实际需求调整评估标准

🧪 测试说明

要测试这个功能，请：

  1. 打开营销任务画布页面
  2. 添加一些节点（开始节点、分流节点、结束节点等）
  3. 切换布局方向（从 TB 到 LR 或从 LR 到 TB）
  4. 打开浏览器开发者工具的控制台
  5. 查看详细的重绘总结日志输出

🔍 查看日志

在浏览器中按 F12 打开开发者工具，切换到 Console 标签页，即可看到详细的重绘总结日志。
