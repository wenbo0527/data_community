# Vue 3 开发规范指南

本指南旨在规范社区前端项目中的 Vue 3 开发流程，提升代码质量和可维护性。

## 1. 组件开发规范

- **Composition API**: 统一使用 `<script setup>` 语法。
- **命名规范**: 
  - 组件文件采用 PascalCase 命名，例如 `UserCard.vue`。
  - 组件名称应具有描述性。

## 2. 状态管理

- 优先使用 `Vuex` 或本地响应式状态。
- 对于跨组件状态，确保逻辑清晰。

## 3. 样式处理

- 使用 Scoped CSS 避免全局样式冲突。
- 遵循 CSS BEM 命名规范。
