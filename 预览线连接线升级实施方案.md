# 预览线连接线升级实施方案

## 项目概述

基于当前 `UnifiedEdgeManager` 版本，实施预览线和连接线的统一管理升级，包含测试案例的全面更新、相关组件的替换、以及需要删除和备份的内容策略。

## 1. 当前系统分析

### 1.1 核心组件现状

#### 主要管理器
- **UnifiedEdgeManager.js**: 已实现统一边管理，包含预览线和连接线的完整生命周期管理
- **PreviewLineSystem.js**: 当前预览线系统的主要入口点
- **PreviewLineManager.js**: 核心预览线管理器
- **PreviewLineService.js**: 预览线服务接口

#### 依赖组件
- **PreviewLineRenderer.js**: 预览线渲染器
- **PreviewLineValidator.js**: 预览线验证器
- **PreviewLineConfigManager.js**: 预览线配置管理器
- **usePreviewLine.js**: 预览线 Composable

### 1.2 使用场景分析

2. **TaskFlowCanvasRefactored.vue** (800+ 行引用)

4. **测试文件** (20+ 个测试文件)

### 5.2 核心迁移阶段 (第2-3周)

#### 第一步：更新主要画布组件
```javascript
// TaskFlowCanvas.vue 更新示例
// 原代码
import { PreviewLineSystem } from '../../../../utils/preview-line/PreviewLineSystem.js'
let previewLineSystem = null

// 新代码
import { UnifiedEdgeManager } from '../../../../utils/unified-edge-manager/UnifiedEdgeManager.js'
let unifiedEdgeManager = null

// 初始化更新
const initializePreviewSystem = async () => {
  if (!unifiedEdgeManager) {
    unifiedEdgeManager = new UnifiedEdgeManager({
      graph: graph.value,
      layoutEngine: window.layoutEngine
    });
    
    await unifiedEdgeManager.initialize();
    window.unifiedEdgeManager = unifiedEdgeManager;
  }
}
```

#### 第二步：更新方法调用
```javascript
// 方法调用映射更新
// 原调用
await previewLineSystem.createUnifiedPreviewLine(node)
await previewLineSystem.deletePreviewLine(previewLineId)
const branches = previewLineSystem.getNodeBranches(node)

// 新调用
await unifiedEdgeManager.createPreviewLine(node)
await unifiedEdgeManager.removePreviewLine(previewLineId)
const branches = unifiedEdgeManager.getNodeBranches(node)
```

#### 第三步：更新全局引用
```javascript
// 全局对象更新
// 原代码
window.previewLineSystem = previewLineSystem

// 新代码
window.unifiedEdgeManager = unifiedEdgeManager
// 保持向后兼容
window.previewLineSystem = {
  ...unifiedEdgeManager,
  // 添加兼容性方法
  createUnifiedPreviewLine: (node) => unifiedEdgeManager.createPreviewLine(node),
  deletePreviewLine: (id) => unifiedEdgeManager.removePreviewLine(id)
}
```
