## 5. 页面设计规范

### 5.1 通用设计模式
- **面包屑导航**: 展示当前位置，便于用户导航
- **操作按钮区域**: 统一放置在页面头部右侧
- **标签页 (Tabs)**: 用于组织不同类别的信息
- **卡片式布局**: 使用 `<a-card>` 组件组织相关内容
- **描述列表**: 使用 `<a-descriptions>` 展示键值对信息

### 5.2 状态标签规范
| 状态 | 颜色 | 含义 |
|-----|------|------|
| 草稿 (draft) | gray | 尚未完成编辑 |
| 待审核 (pending) | orange/blue | 等待审核中 |
| 有效/上线 (active/online) | green | 正常可用 |
| 无效/下线 (inactive/offline) | red | 不可用 |
| 已过期 (expired) | lightgray | 已过期废弃 |

### 5.3 数据类型标签规范
| 类型 | 说明 |
|-----|------|
| 数值型 | numerical - 连续或离散的数值数据 |
| 分类型 | categorical - 有限数量的分类数据 |
| 文本型 | text - 文本字符串数据 |
| 时间型 | datetime/time - 日期和时间数据 |
| 布尔型 | boolean - 是/否类型数据 |

## 6. 数据流向和关联

### 6.1 资产关联关系
```
表 (Table)
  └── 包含多个字段 (Field)
  └── 被指标引用 (Metric)
  └── 被变量引用 (Variable)
  └── 被特征引用 (Feature)

指标 (Metric)
  └── 引用多个变量 (Variable)
  └── 归属特定场景 (Scene)

变量 (Variable)
  └── 引用多个表字段 (Field)
  └── 归属特定业务域 (Domain)
  └── 被模型使用 (Model)

特征 (Feature)
  └── 依赖多个表 (Table)
  └── 可能关联模型注册信息 (PlatformModel)
```

### 6.2 详情页导航关系
- 表详情页 → 可跳转至相关指标/变量/特征详情页
- 指标详情页 → 可查看血缘关系、依赖的变量和表
- 变量详情页 → 可查看上下游血缘、使用场景、版本历史
- 特征详情页 → 可查看模型注册信息、数据来源

## 7. Mock 数据支持

所有详情页使用 Mock 数据进行展示，Mock 数据定义在以下位置：
- **表数据**: `src/mock/shared/metadata-store.ts`
- **指标数据**: `src/mock/metrics.ts` 或 `src/mock/metrics-data.ts`
- **变量数据**: `src/store/modules/variable.ts` (Pinia Store)
- **特征数据**: `src/api/offlineModel.ts`

## 8. 备注

- 所有详情页均支持响应式设计
- 表详情页相对简单，主要展示基本信息和字段列表
- 指标详情页注重口径信息和SQL代码展示
- 变量详情页注重数据质量和血缘关系
- 特征详情页相对简洁，特殊之处在于模型输出类型的特征会显示额外的模型注册信息
