# 问题编号：001
## 日期：2026-02-24

### 项目背景
项目是一个基于Vue的社区管理系统，包含模型离线分析模块。用户希望在新建模型页面（/model-offline-analysis/model-register/create）中移除“算法框架”和“超参数”相关的内容。

### 问题分析
用户反馈需要删除“算法框架”和“超参数”输入项。
经过分析代码 `/src/pages/risk/model-offline-analysis/modelRegister/create.vue`，发现：
1.  “算法框架”对应表单字段 `framework`，必填项。
2.  “超参数”对应表单字段 `parameters`，非必填项，JSON格式。

这两个字段在前端表单中显示，并且在提交时会发送给后端。
需要做出的修改：
1.  从模板（template）中移除对应的 `<a-form-item>` 和布局容器。
2.  从数据模型（script setup）中移除 `framework` 和 `parameters` 的定义。
3.  移除相关的验证规则（rules）。
4.  移除数据回显和提交时的相关处理逻辑。

### 解决方案
1.  **修改 Template**：
    *   移除 `field="framework"` 的 `<a-form-item>` 及其父级 `<a-col>`。
    *   移除 `field="parameters"` 的 `<a-form-item>` 及其父级 `<a-col>`。
    *   调整 `<a-row>` 布局，确保剩余表单项（如“模型类型”、“版本”）布局合理。
2.  **修改 Script**：
    *   在 `form` ref 中删除 `framework` 和 `parameters` 属性。
    *   在 `rules` 对象中删除 `framework` 的验证规则。
    *   在 `handleServiceChange` 方法中，移除对 `framework` 的赋值。
    *   在 `handleSubmit` 和 `handleSave` 方法中，移除对 `parameters` 的 JSON 解析逻辑。
