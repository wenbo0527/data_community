# 数据发现平台项目说明文档

## 1. 产品定位与价值主张
让业务找得到、看得懂、用得动所有数据资产的数据探索中心。

### 1.1 产品愿景
通过统一的数据发现入口，降低数据使用门槛，提升数据发现效率，减少数据重复建设成本，将“数据找人”变成“人找数据”

### 1.2 核心价值主张
核心价值量化表：
| 维度     | 现状痛点                 | 平台解法           |
| ------ | -------------------- | -------------- |
| **发现** | 分散在 7+ 系统，平均找数 2.5 h | 统一搜索 +  快速导览   |
| **理解** | 字段含义靠口口相传            | 资产详情标准化 + 质量评分 |
| **使用** | 重复建表、口径不一            | 注册即复用、指标口径集中治理 |
| **治理** | 注册、变更、下线无闭环          | 资产全生命周期线上化     |

### 1.3 目标用户画像
核心用户画像：
| 角色              | 高频场景        | 关键诉求            | 
| --------------- | ----------- | --------------- | 
| 业务分析师 / 运营 / 风控 | 日常跑数、策略迭代、风控模型调优、业务监控报表 | 快速定位指标、变量、外部数据源，理解数据血缘关系，评估数据质量 | 
| 数据开发 / 数据管理     | 注册、变更、下线资产，数据质量监控，权限管理 | 流程在线化减少手工操作，口径统一避免歧义，减少业务答疑工作量 | 
| 数据科学家 / 算法工程师   | 特征工程、模型训练、算法优化、A/B测试分析 | 快速发现高质量特征| 
| 通用角色           | 权限申请、资产共享 | 统一申请减少沟通，数据资产权限管理 |


### 1.4 业务目标与成功指标
【KPI指标：月活跃用户>500人、数据发现时间<2分钟、数据资产覆盖率>90%、用户满意度>85%、数据重复查询减少60%】

## 2. 用户研究与场景分析

### 2.1 用户痛点与场景故事

| 场景故事          | 当前痛点                    | 平台解法                        | 期望用户反馈                  | 
| ------------- | ----------------------- | --------------------------- | ---------------------- | 
| **风控专员找逾期数据** | 问 5 个人、跑 3 次 SQL、字段含义不清 | "逾期"关键词一键搜索 → 质量评分 → 直接申请权限 | 数据查找时间从小时级缩短至分钟级 | 
| **产品经理做营销**   | 不知道哪些行为标签可用，重复埋点        | 指标地图 + 血缘图谱 → 直接复用          | 减少重复开发，标签复用率提升70%         | 
| **算法工程师找特征**   | 不同业务或团队存在哪些表不知道，共享依赖线下脚本传播        | 特征仓库 → 质量评分 → 快速验证          | 特征发现效率提升，模型迭代周期缩短50%    |
| **客户运营找用户画像**   | 用户标签分散在7个系统，查一个用户要开8个页面        | 客户360统一视图 → 一键查询 → 全量标签展示          | 客户洞察效率提升，查询时间缩短80%    |
| **风险模型调优**   | 模型特征缺失，数据质量差，模型效果不稳定        | 特征质量评分 → 血缘追溯 → 质量预警          | 模型稳定性显著提升，特征可用率提高    |
| **内部培训新员工**   | 数据部门新员工找不到数、不知道怎么用、工作上手慢，培训流程繁琐、效果不佳        | 运营大屏 → 技术大屏 → 质量监控        | 培训效果与学习效率显著提升    |

### 2.3 用户旅程地图
【需求产生→平台搜索→数据发现→质量评估→权限申请→数据使用→效果反馈→资产优化】

## 3. 产品功能架构

### 3.1 整体功能架构图
【统一入口层：全局搜索+个性化推荐
业务功能层：资产总览+统一搜索+全量数据+数据资产+数据注册
技术支撑层：元数据管理+权限控制+数据质量+审计日志】
导览：资产总览
核心查询页：统一搜索（全局）
资产分类列表页：全量数据、指标地图、征信变量、外部数据
资产管理页：外部数据管理、指标管理、表管理、其他非体系化的使用批量资产管理的方式上传统一管理
核心数据资产：数据表（全量表、结构化、非结构化）、外部数据、业务指标、变量（结构化、非结构化）

### 3.2 功能列表和介绍

### 3.2.1 资产总览 (Asset Overview)

**对应场景**：管理层看数据资产全局视图

资产总览模块帮助用户快速了解企业数据资产全貌，解决"数据在哪里、有多少、质量如何"的核心问题。

**核心功能**：
- 数据资产全景视图：一眼看清企业有多少数据表、指标和外部数据源
- 数据热度排行榜：发现哪些数据被频繁使用，哪些数据被闲置
- 质量评分体系：为每个数据资产打上质量标签，避免使用脏数据
- 资产增长趋势：追踪数据资产的动态变化，及时发现新增数据

**用户价值**：
从"找不到数据"到"一眼看清数据全貌"，让数据资产变成可视化的业务资源

**核心代码路径**：
```
src/pages/management/data-map/
├── index.vue                    # 资产总览主页面
└── components/
    ├── AssetOverview.vue       # 资产概览组件
    ├── AssetStats.vue          # 资产统计组件
    └── AssetTrendChart.vue     # 资产趋势图表组件
```

### 3.2.2 统一搜索 (Unified Search)

**对应场景**：风控专员找逾期数据、产品经理做营销策略、客户运营找用户画像

统一搜索模块解决"数据在哪里"的核心痛点，让业务人员像用百度一样找数据。

**核心功能**：
- 一站式数据搜索：输入"用户逾期"就能找到所有相关数据表和指标
- 智能联想搜索：输入"营销"自动推荐用户标签、活动效果等数据
- 个人工作台：收藏常用数据，建立个性化数据清单
- 业务场景导航：通过"获客-风控-运营"等业务流程快速定位数据

**用户价值**：
从"问人要数据"到"30秒找到数据"，业务人员可以自助获取所需数据，不再依赖技术人员

**核心代码路径**：
```
src/pages/discovery/search/
├── index.vue                    # 统一搜索主页面
├── components/
│   ├── SearchResults.vue       # 搜索结果组件
│   ├── BusinessProcessFlow.vue # 业务流程展示组件
│   ├── CollectionManager.vue   # 数据集合管理组件
│   └── QuickAccessTables.vue   # 常用表快速访问组件
```

### 3.2.3 全量数据 (Full Data Access)

**对应场景**：算法工程师找特征、内部培训新员工

全量数据模块像数据地图一样，帮助用户系统性地了解企业有哪些数据，数据之间有什么关系。

**核心功能**：
- 数据地图导航：像超市货架一样分类展示所有数据表
- 智能标签系统：通过"用户行为"、"风控指标"等标签快速筛选数据
- 数据血缘追踪：点击任意数据表，清晰看到数据来源和去向
- 一键数据探查：无需写SQL，直接预览数据内容和统计信息

**用户价值**：
从"盲人摸象"到"一览全局"，技术人员和业务人员都能快速理解企业数据全貌，新员工培训时间从1周缩短到1天

**核心代码路径**：
```
src/pages/discovery/data-map/
├── index.vue                    # 全量数据主页面
├── TableList.vue               # 数据表列表页面
├── TableDetailPage.vue         # 表详情页面
└── components/
    ├── CollectionGrid.vue       # 集合网格视图
    ├── CollectionList.vue       # 集合列表视图
    └── SearchFilters.vue        # 搜索过滤组件
```

### 3.2.4 数据资产 (Data Assets)

**对应场景**：风险模型调优、营销策略验证

数据资产模块让数据资产变得"活"起来，支持业务快速迭代。

**核心功能**：
- 指标工厂：像搭积木一样定义新的业务指标，无需开发介入
- 版本时光机：每个指标的变更都有记录，随时回滚到历史版本
- 影响分析：修改一个指标前，自动显示会影响哪些报表和模型
- 权限管控：不同角色看到不同的数据资产，确保数据安全

**用户价值**：
从"每次改指标都要找开发"到"业务人员自助配置"，模型迭代周期从2周缩短到2天

**核心代码路径**：
```
src/pages/management/data-map/
├── index.vue                    # 数据资产主页面
└── components/
    ├── TableDetail.vue         # 数据表详情组件
    ├── MetricDetail.vue        # 指标详情组件
    ├── AssetTree.vue           # 资产树形导航组件
    └── AssetDrawer.vue         # 资产详情抽屉组件
```

### 3.2.5 数据注册 (Data Registration)

**对应场景**：新业务数据源接入

数据注册模块让新数据接入变得像填写快递单一样简单，告别复杂的技术配置。

**核心功能**：
- 傻瓜式注册向导：像填写表单一样完成数据源接入，无需技术背景
- 一键质量体检：新数据接入时自动检测数据质量，提前发现潜在问题
- 审批工作流：数据接入需要经过标准化审核，确保数据质量和安全
- 批量导入神器：支持Excel批量注册，一次性导入大量数据表

**用户价值**：
从"需要技术人员花1周接入"到"业务人员30分钟自助完成"，让数据接入不再是技术瓶颈

**核心代码路径**：
```
src/pages/management/
├── table-management/
│   └── index.vue               # 数据表注册页面
├── metric-management/
│   └── index.vue               # 指标管理页面
├── batch-asset-management/
│   └── index.vue               # 批量资产管理页面
└── external-data-management/
    └── index.vue               # 外部数据管理页面
```
