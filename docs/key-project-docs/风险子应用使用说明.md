# 风险子应用使用说明（开发阶段）

## 开关与地址
- VITE_RISK_EXTERNAL：开启后 /risk 路由改为外部子应用挂载（iframe），默认关闭。
- VITE_RISK_EXTERNAL_URL：子应用地址，默认为 http://localhost:5174。

## 使用步骤
- 本地开发：在根目录 .env.development 设置 `VITE_RISK_EXTERNAL=true`，并确保子应用在 5174 端口可访问。
- 路由：访问主应用的 `/risk/...` 将在同一路径下加载子应用页面，参数与查询会透传。

## 迁移建议
- 先按页面模块迁移到子应用，保持命名路由与路径一致。
- 公共组件与工具抽取到 shared 库，使用别名统一引用。

## 回退与兼容
- 关闭开关 `VITE_RISK_EXTERNAL=false` 后，主应用恢复原有 /risk 路由与页面。
