# 数据服务模型管理（Data Models）产品需求文档 (PRD)

## 1. 文档基本信息

- **文档名称**：数字社区平台-数据服务模型管理需求文档
- **版本信息**：v1.0.0
- **关联路径**：`src/pages/management/data-models`
- **业务归属**：数据管理（管控端） -> 数据服务管理 -> 数据服务模型管理
- **编写人**：AI需求分析专家
- **生效状态**：已生效

## 2. 需求背景与目标

**背景**：
在企业的数据管理体系中，为了满足业务端灵活多变的数据查询、分析报告、特征计算和数据清洗需求，往往需要将常用的查询逻辑和处理脚本进行沉淀和复用。原有的零散脚本管理模式缺乏统一的生命周期管理、参数化配置和执行历史追溯。

**目标**：
- 提供统一的数据查询和处理脚本（模型）的管理平台。
- 支持 SQL 和 Python 两种主要语言类型，适配不同的处理场景。
- 建立模型生命周期管理（草稿、已发布、已归档）。
- 提供可视化的参数配置能力，支持一键执行并留存执行历史与统计指标。
- 赋能业务人员和数据开发，提高数据加工流转效率和资产复用率。

## 3. 用户角色与使用场景

- **数据开发/分析师**：编写 SQL/Python 脚本，配置输入/输出参数，进行在线测试和调试，最后发布模型。
- **业务运营人员**：查找已发布的“分析报告模板”或“明细数据下载”模型，输入业务参数（如时间范围、用户类型），一键执行并获取数据结果。
- **数据管理员**：监控各模型的执行情况（成功率、平均耗时），清理和归档过期模型。

## 4. 核心名词解释

- **数据模型 (Data Model)**：一段带有参数化配置的 SQL 查询或 Python 处理脚本，用于特定的业务场景。
- **使用场景 (UseCase)**：对模型的业务分类，包括：明细数据下载、分析报告模板、外数样本表、校验规则、数据分析、数据清洗、特征工程等。
- **语言类型 (Language)**：脚本的执行环境，目前支持 SQL 和 Python。
- **入参 (Input Parameters)**：执行模型时需要外部传入的动态变量（如 `start_date`, `limit`）。
- **执行历史 (Execution History)**：模型每次被执行的记录，包含耗时、状态、返回行数等信息。

## 5. 核心功能模块

### 5.1 数据模型列表 (DataModelsList.vue)
**功能描述**：展示所有数据模型，支持搜索和快捷操作。
- **筛选条件**：
  - 模型名称（模糊搜索）
  - 使用场景（下拉选择）
  - 状态（草稿、已发布、已归档）
  - 语言类型（SQL、Python）
- **列表字段**：
  - 模型名称与描述（带有可点击跳转详情的链接）
  - 使用场景（带颜色标签）
  - 语言类型（带颜色标签）
  - 状态（带颜色标签）
  - 版本号
  - 更新时间
  - 管理人（头像+名称）
  - 操作列（查看、编辑、复制、删除）
- **核心操作**：
  - **新增模型**：跳转至新建表单页。
  - **复制模型**：快速创建一个配置相同的副本模型（状态默认为草稿）。

### 5.2 模型表单配置 (DataModelsForm.vue)
**功能描述**：用于新建和编辑数据模型，支持基础信息和代码/参数配置。
- **基本信息配置**：
  - 模型名称（必填，2-50字符）
  - 模型描述（必填，限200字符）
  - 使用场景（必填下拉）
  - 语言类型（必填，切换语言时代码编辑器将自动变更语法高亮）
  - 管理人
- **模型配置 (代码与参数)**：
  - **代码编辑器**：集成 Monaco Editor，提供 SQL/Python 语法高亮和基本的代码检查功能（如SQL的SELECT/FROM关键字检查）。
  - **参数定义**：动态增删改的参数列表。
    - 参数字段：参数名、类型（string/number/date/boolean等）、是否必填、默认值、描述。
    - 支持通过上下箭头调整参数顺序。
  - **执行配置**：
    - 超时时间 (Timeout)：默认 300秒。
    - 最大内存限制 (Max Memory)：默认 1024MB。
- **保存机制**：
  - **保存草稿**：状态变更为 `draft`，如果是新建操作，保存后路由替换为编辑模式。
  - **创建/更新模型**：进行完整的表单校验，提交后状态变更为 `published` 并返回列表。

### 5.3 模型详情与执行 (DataModelsDetail.vue)
**功能描述**：查看模型详细配置、在线执行测试、查看历史记录。
- **信息展示**：
  - 顶部栏：标题、状态标签、复制/编辑/执行操作按钮。
  - 基础信息卡片：展示创建/更新时间、版本、语言、场景、超时等属性。
  - 参数配置表：表格形式展示定义的入参列表。
  - 代码查看器：只读模式的代码展示，支持“一键复制”和“下载文件”。
- **执行功能 (Execute)**：
  - 点击“执行”按钮触发模型运行。
  - 弹出结果模态框，展示执行状态（成功/失败）、耗时、执行日志（Log）和数据结果片段（Result）。
- **执行历史与统计**：
  - 统计卡片：总执行次数、成功次数、失败次数、平均耗时。
  - 历史记录列表：展示最近的执行时间、执行人、状态、耗时和影响行数。点击可查看单次执行的详情日志。

## 6. 数据结构与接口定义 (Mock API)

模块依赖于 `src/mock/data-models.ts` 提供的 Mock 数据服务。

### 6.1 核心数据结构 (DataModel)
```typescript
interface DataModel {
  id: string;
  name: string;
  useCase: 'download' | 'report' | 'external_data_sample' | 'validation_rule' | 'data_analysis' | ...;
  language: 'sql' | 'python'; // 在表单中使用的字段名
  manager: string;
  status: 'draft' | 'published' | 'archived';
  version: string;
  description: string;
  code: string;
  parameters: Array<{
    name: string;
    type: 'string' | 'number' | 'boolean' | 'date';
    defaultValue: any;
    required: boolean;
    description: string;
  }>;
  timeout: number;
  maxMemory: number;
  createdAt: string;
  updatedAt: string;
}
```

### 6.2 接口列表
| 接口描述 | 方法 | URL 路径 |
|---|---|---|
| 获取模型列表 | GET | `/api/data-models` |
| 获取模型详情 | GET | `/api/data-models/:id` |
| 创建数据模型 | POST | `/api/data-models` |
| 更新数据模型 | PUT | `/api/data-models/:id` |
| 删除数据模型 | DELETE | `/api/data-models/:id` |
| 复制数据模型 | POST | `/api/data-models/:id/copy` |
| 执行数据模型 | POST | `/api/data-models/:id/execute` |
| 获取执行历史 | GET | `/api/data-models/:id/executions` |
