```bash

node scripts/start-all.js

# 方式3：concurrently直接启动
npx concurrently \
  "npm run dev:portal" \
  "npm run dev:risk" \
  "npm run dev:mkt" \
  "npm run dev:dex" \
  "npm run dev:dmt" \
  "npm run dev:admin"
```

## 🔧 实施步骤

### Step 1: 创建根目录配置（15分钟）

```bash
# 进入数据社区根目录
cd ~/Documents/project/data_community

# 创建package.json（内容见方案文档）
touch package.json

# 创建.env（内容见方案文档）
touch .env

# 安装根依赖
npm install
```

### Step 2: 创建启动脚本（30分钟）

```bash
# 创建scripts目录
mkdir -p scripts

# 创建启动脚本（内容见方案文档）
touch scripts/start-all.js
touch scripts/check-ports.js
touch scripts/kill-all.js
```

### Step 3: 修改各子应用端口（30分钟）

修改各子应用的 `vite.config.ts`：

```typescript
// apps/risk-app/vite.config.ts
export default defineConfig({
  base: '/risk/',
  server: {
    port: 8001,  // 从5176改为8001
    host: '0.0.0.0',
  },
  // ...其他配置
})
```

同理修改：
- mkt-app: 8002
- dex-app: 8003
- dmt-app: 8004
- admin-app: 8005

### Step 4: 更新portal-shell注册配置（15分钟）

修改 `portal-shell/src/App.vue` 中的子应用注册：

```typescript
const apps: AppConfig[] = [
  { 
    name: '数字风险域 (RISK)', 
    entry: 'http://localhost:8001/',  // 从5176改为8001
    // ...其他配置
  },
  { 
    name: '营销域 (MKT)', 
    entry: 'http://localhost:8002/',  // 从5177改为8002
    // ...其他配置
  },
  // ...其他子应用同理
]
```

### Step 5: 验证测试（30分钟）

```bash
# 一键启动所有服务
npm run dev:all

# 检查各端口是否可访问
npm run check:ports

# 访问主框架
open http://localhost:8000

# 测试各子应用切换
```
